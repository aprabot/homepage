Optimization Recommendations

Page 5 of 7

Immediate Wins (Day 1)

1 — Sparse Filtering Before SageMaker Inference

Before every batch run, pre-filter the 400M SKU×zip combinations to only those with non-zero demand history. Typically 5–20% are active, reducing inference input from 400M to 20–80M records.

# Pre-inference filter (Athena query, runs in seconds) SELECT sku_id, zipcode FROM demand_history WHERE run_date >= CURRENT_DATE - INTERVAL '13' WEEK AND total_units > 0 GROUP BY sku_id, zipcode -- Result: 20–80M rows instead of 700M → 70–95% cost reduction on inference

Estimated savings: $9–80/client/mo on SageMaker.

2 — Athena Partition Strategy

Partition S3 data as company={id}/sku_prefix={2-char}/run_date={date}/. Every Athena query filters by company and date, pruning 99%+ of the dataset.

Without partitioning, a single query scanning 440 GB costs $2.20. With partitioning scanning 500 MB, it costs $0.0025 — 880× cheaper.

3 — Share Bedrock Knowledge Base Across Clients

The OpenSearch Serverless cluster runs at a minimum of 7 OCUs regardless of client count. With one shared KB domain (with per-client namespace isolation via metadata filters), the $700/mo fixed cost is divided across all clients.

Medium-Term (3–6 Months)

4 — Replace AWS Transfer Family with S3 Pre-Signed URLs

Transfer Family costs $0.30/hr always-on ($216/mo). For partners that can use REST APIs, pre-signed S3 URLs eliminate the SFTP server entirely. Keep Transfer Family only for partners requiring SFTP compliance.

Estimated savings: $150–200/mo on shared infrastructure.

5 — SageMaker Savings Plans

Once inference patterns are stable, commit to a 1-year SageMaker Savings Plan. This provides ~40% discount on on-demand price (effectively ~15% additional discount vs Spot for covered usage).

6 — S3 Intelligent-Tiering Lifecycle Policy

Historical data older than 30 days rarely needs immediate access. Intelligent-Tiering automatically moves it to Infrequent Access ($0.0125/GB) and Archive ($0.004/GB) tiers, reducing effective storage cost by 40–80% on the historical dataset.

AWS Services

TermDefinition
Amazon S3
Simple Storage Service		 Object storage service used to store raw data lake files, compressed Parquet datasets, forecast outputs, and model artifacts. Billed per GB stored and per request.
S3 Intelligent-Tiering Automatic S3 storage class that moves objects between Frequent Access ($0.023/GB), Infrequent Access ($0.0125/GB), and Archive tiers ($0.004/GB) based on access patterns. Reduces cost on ageing historical data with no performance penalty on retrieval.
Amazon SageMaker
Batch Transform		 Managed ML service used to run large-scale batch inference jobs. Batch Transform reads input data from S3, distributes inference across a fleet of instances, and writes predictions back to S3. Billed per instance-hour while the job runs.
AWS Glue Serverless ETL (Extract, Transform, Load) service built on Apache Spark. Used to ingest partner data, compute lag and rolling features, and write partitioned Parquet files to S3. Billed per DPU-hour consumed.
Amazon Athena Serverless, interactive SQL query engine that runs directly over files stored in S3. Requires no cluster setup. Billed at $5 per TB of data scanned — partition pruning is critical to keeping costs near zero.
Amazon Bedrock Fully managed service for accessing foundation models (Claude, Titan, Llama, etc.) via API. APRABot uses Bedrock to power the multi-agent system (Query, Orchestrator, Executor, Analysis agents) and the Knowledge Base. Billed per input/output token.
Amazon Bedrock Knowledge Base Managed RAG (Retrieval-Augmented Generation) service that indexes documents into a vector store (OpenSearch Serverless) and retrieves context at query time. Stores forecast schema, S&OP glossary, SKU taxonomy, and model metadata.
Amazon OpenSearch Serverless Serverless vector search engine backing the Bedrock Knowledge Base. Billed per OpenSearch Compute Unit (OCU) per hour with a minimum of 2 OCUs for indexing and 2 OCUs for search. Fixed cost regardless of query volume.
Amazon EventBridge Serverless event bus and scheduler. Used to trigger the weekly SageMaker inference pipeline on a cron schedule. Essentially free at this usage level.
AWS Transfer Family Managed SFTP/FTPS/FTP service that allows partner companies to deliver data files directly into S3 using standard file-transfer protocols. Billed at $0.30/hour for the server endpoint plus $0.04/GB transferred.
Amazon Cognito User identity and access management service. Handles planner authentication (user pools), MFA, SSO/SAML federation, and per-tenant isolation. Billed per Monthly Active User (MAU) above the 50,000 free tier.
Amazon API Gateway + Lambda REST API layer and serverless compute. API Gateway routes incoming user queries from the portal to the Orchestrator Agent and returns analysis responses. Lambda functions handle routing logic. Both near-free at <10K calls/month.
Amazon CloudFront Content delivery network (CDN) used to serve the APRABot portal globally with low latency. Caches static assets at edge locations. Billed per GB transferred and per HTTPS request.
Amazon CloudWatch Monitoring and observability service. Collects logs from Lambda, SageMaker, Glue, and Bedrock; triggers alerts on pipeline failures; tracks inference latency and cost metrics.

Technical Terms

TermDefinition
Time Series A sequence of values recorded at regular time intervals for a single (SKU, Zipcode) combination. In this model, each time series contains 104 weekly demand observations spanning 2 years. The full client dataset contains up to 400 million distinct time series.
SKU
Stock Keeping Unit		 A unique identifier for a distinct product variant (e.g. a specific size, colour, and pack format of a product). Each client has up to 20,000 SKUs. Forecasts are generated at the SKU level, disaggregated by zipcode.
Zipcode A US postal code used as the geographic demand unit. Each client covers up to 20,000 zipcodes. Combining 20K SKUs × 20K zipcodes produces a 400M-cell demand matrix.
Parquet A columnar binary file format optimised for analytical workloads. Parquet stores data column-by-column (versus row-by-row in CSV), enabling Athena and Spark to read only the columns needed for a query. Combined with Snappy compression, achieves 5–10× size reduction over raw CSV.
Snappy Compression A fast, lossless compression algorithm used with Parquet files. Prioritises compression/decompression speed over maximum ratio — typically 5–10× for time series data. Results in smaller S3 files and faster Athena scans.
Partition Pruning An Athena / Spark optimisation where queries skip entire S3 prefixes (partitions) that cannot match the WHERE clause. For example, partitioning by run_date means a query for last week's forecasts scans only 1/104th of the dataset, reducing cost by ~99%.
Sparse Filtering Pre-inference step that removes SKU×zipcode combinations with zero sales history from the batch inference input. Reduces the 400M-series full matrix to the 5–20% of combinations that have actual demand, cutting SageMaker compute cost by up to 80%.
Spot Instances Spare AWS EC2/SageMaker capacity offered at 60–70% discount versus on-demand pricing. Can be interrupted with 2 minutes' notice if AWS needs the capacity back. Safe for SageMaker Batch Transform because jobs checkpoint progress and resume automatically on interruption.
ETL
Extract, Transform, Load		 The data pipeline process of extracting raw partner data (via SFTP or REST), transforming it (cleaning, normalising, computing features), and loading the result into S3 in the format required by the ML model.
DPU
Data Processing Unit		 AWS Glue billing unit. One G.1X DPU provides 4 vCPUs and 16 GB RAM. Billed at $0.44/DPU-hour. A typical Glue ETL job uses 10 DPUs, consuming 4.4 DPU-hours in a 26-minute run.
OCU
OpenSearch Compute Unit		 OpenSearch Serverless billing unit representing a unit of indexing or search capacity. Minimum 2 OCUs for indexing + 2 OCUs for search (4 OCUs total) at $0.24/OCU-hour = ~$700/month fixed cost regardless of usage, shared across all clients.
XGBoost
eXtreme Gradient Boosting		 A gradient-boosted decision tree algorithm widely used in tabular ML. One of two models in APRABot's ensemble. Scores ~100M records per hour per ml.m5.12xlarge instance. Outputs demand point estimates.
LightGBM
Light Gradient Boosting Machine		 A fast gradient boosting framework from Microsoft. The second model in APRABot's ensemble. Complementary to XGBoost — together they reduce forecast error versus either model alone. Ensemble predictions are blended (e.g. average or stacked) before storage.
Ensemble A forecasting strategy that combines predictions from multiple models (here XGBoost + LightGBM) to produce a final forecast. Ensembles consistently outperform single models by reducing variance and bias, at the cost of running both models in parallel.
RAG
Retrieval-Augmented Generation		 A technique where an LLM's response is grounded in retrieved documents rather than relying solely on its training data. The Bedrock Knowledge Base retrieves relevant forecast schema, glossary, and SKU context before the Analysis Agent generates its response, reducing hallucinations.
WAPE
Weighted Absolute Percentage Error		 The primary forecast accuracy metric used in APRABot. Defined as sum(|actual − forecast|) ÷ sum(actual). Volume-weighted, so high-volume SKUs drive the metric more than low-volume ones. Lower is better; industry benchmark is 20–35% for weekly SKU-level demand.

Business Terms

TermDefinition
Gross Margin Revenue minus cost of goods sold (COGS), expressed as a percentage of revenue. For SaaS, COGS is primarily cloud infrastructure. A 85% gross margin means $85 of every $100 in revenue remains after paying AWS bills. Top-quartile AI SaaS companies target 75–85%.
SaaS
Software as a Service		 A subscription software delivery model where the vendor hosts and maintains the product. Customers pay a recurring monthly or annual fee rather than a one-time licence. APRABot operates as a SaaS with per-client monthly pricing.
MAU
Monthly Active User		 A unique user who logs into the platform at least once in a calendar month. Cognito bills per MAU above the 50,000 free tier. At 5–20 users per client, Cognito cost is negligible.
S&OP
Sales & Operations Planning		 A cross-functional business process that aligns supply chain, finance, and sales around a single demand plan. APRABot's portal supports the S&OP cycle with AI-generated commentary, scenario planning, and override workflows.
EDP
Enterprise Discount Program		 A private pricing agreement between AWS and customers who commit to a minimum annual spend (typically $500K+). EDP typically provides 10–25% discount across all AWS services, applied on top of Spot and Savings Plan discounts.
Savings Plans AWS commitment-based discount model. SageMaker Savings Plans offer ~40% off on-demand compute in exchange for a 1- or 3-year commitment to a minimum hourly spend. Recommended once inference patterns are stable and predictable.
